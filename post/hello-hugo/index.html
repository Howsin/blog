使用hugo於github上自架blog

September 7, 2021

隨著接觸到的技術越來越多，每次看過資料總是只有當下有記憶，只有那些常常用到的技術記憶相對深刻，其餘的就算解決過再遇到還是必須要再查一次，這時候就只能憑著印象去重頭摸索了，如果能有一個自己的技術blog，吸收了這些知識後寫下自己內化過的理解，一方面可以幫助遇到同樣問題的朋友，一方面也可以讓自己印象加深，看起來是個完美的解決方案。

其實想經營一個技術blog的念頭一直都在，但遲遲無法下定決心，總是寫個一兩篇就無限停更，找不到一個維持住寫作熱情的動力，或許是因為以往開始這件事太輕鬆，找個主流blog網站申請個帳號就可以開始，不花力氣的事就不會太重視，這才是真正的原因，絕對不是我太懶。

以往寫blog都使用線上服務，雖然功能都很完善，但總找不到一個所有要求都符合自己需求的平台，對於開發者來說，用別人的系統多少會有種綁手綁腳的感覺，相較於使用介面，我更喜歡直接面對程式碼，只要我願意花時間去看code，想要什麼功能都能自己想辦法弄出來，這種踏實感在我卡關並懷疑自己是不是時間太多的時候，可以起到很好的激勵效果。

Hugo

與曾經使用過的任何內容管理系統不同，不需要後台介面與資料庫，Hugo是一個開放原始碼的框架工具，可以快速生成一個靜態網站，甚至不需要有資訊背景，讓使用者專注在文章撰寫上，因為Hugo本身是用Go開發，在轉換與打包成靜態網站的速度相較於其他工具快上很多。

架站部分，只要選擇任一網頁伺服器，簡單的將打包好的程式碼放上去就好，而也因為是靜態網站，不需要和後端同步資料或是動態渲染頁面資料，部署後直接可以被瀏覽器存取，SEO與速度方面相較於動態網站更佔優勢，靜態頁面也可以方便的部署CDN，加速網站連線速度。

Solution

由於github上提供了免費的靜態檔案伺服器，可以搭配hugo成為自架blog的完整解決方案，未來更新文章與blog設定時也能用git來做控管，整個自架步驟將會照著以下流程：

  1. 安裝hugo
  2. 新建一個網站模板
  3. 挑選theme後下載與套用
  4. 新增文章與測試
  5. git與github上新增repository
  6. 設定github連結頁面
  7. 大功告成檢視結果

以下是本篇中會使用到的環境與工具，其中會需要對Git有最初步的認識與使用經驗。(以下工具皆為跨平台，其他作業系統也可以使用)

  • Local
    • macOS：本篇中使用的作業系統
    • Typora Editor：Markdown編輯器，可自行替換
    • Git：需自行下載安裝
    • Hugo：本篇主要使用的打包工具
    • Go：需自行下載安裝
  • Site
    • Github：需有登入帳號

Start

一、安裝Hugo

  • build from source code

  • 安裝方式有很多種，也可透過官方提供編譯後的binary檔直接使用，詳細可參考這裡。

二、新建一個網站模板
  • 此步驟將會在當前目錄底下建立一個blog資料夾，並於其中建立所需檔案。
三、挑選theme後下載與套用

  • 由於新建好的網站中預設是沒有theme的，這時最快的方法就是到官網中挑選。

  • 依據挑選，不同的theme會大致介紹安裝方式與一些theme本身的設定，詳細請參考theme本身介紹頁面。

  • 下方使用m10c作為本篇範例，在進入上一步驟新增的網站資料夾blog後，依據theme頁面的步驟下載。

  • 此時themes資料夾中會存在下載好的樣式

  • 編輯blog中的 config.toml，加入theme="m10c"

四、新增文章與測試

  • 做完上一個步驟後，已經有了最初步的網站，這時可以先打包測試後觀察結果

  • 這時直接在瀏覽器中輸入 http://localhost:1313/ 觀察測試結果，可以看到輸出後的畫面

  • 繼續調整config.toml設定，並於瀏覽器上觀察設定後的結果

  • 新增文章，至content中開啟後以markdown編輯文章內容

  • 瀏覽器中已經可以在頁面上看到新增的文章

五、git與github新增repository

  • 在Github中新增git repository blog

  • 至public資料夾中新增git repository

  • 本地端repository與上傳至github的設定較為繁瑣，請參考此篇文章

六、設定github連結頁面
  1. 在github上選擇剛剛新增的repository
  2. 選擇Setting
  3. 左方選單中選擇Pages
  4. 右方source中選擇/ (root)，儲存Save
  5. 儲存後至上方顯示的連結Your site is published at https://howsin.github.io/blog/
  6. 檢視畫面，大功告成

使用流程

完成上述的步驟後，未來更新文章依照這些步驟

  1. 使用hugo新增文章
  2. 在Typora Editor中編輯文章
  3. hugo server檢視最終結果
  4. 使用git更新至github
  5. end

心得

雖然過程看起來很繁雜，但等一切設定好之後就很方便了，就是寫文章->push，要調整設定config.toml改完->push，想加什麼功能跟頁面也可以自行修改，畢竟所有code都有，也可以找別人開發的外掛。

終於完成啦～這樣之後就沒理由偷懶了，寫了這篇也花了不少時間，但不知道是不是太久沒寫文章，總覺得堆砌詞句上沒有那麼順暢，沒辦法很好用文字表達意思的感覺，希望未來可以越寫越順～
