使用hugo在github上架設客製化域名的blog

September 7, 2021

Introduction

隨著接觸到的技術越來越多，每次看過資料總是只有當下有記憶，只有那些常常用到的技術記憶相對深刻，其餘的就算解決過再遇到還是必須要再查一次，這時候就只能憑著印象去重頭摸索了，如果能有一個自己的技術blog，吸收了這些知識後寫下自己內化過的理解，一方面可以幫助遇到同樣問題的朋友，一方面也可以讓自己印象加深，看起來是個完美的解決方案。

其實想經營一個技術blog的念頭一直都在，但遲遲無法下定決心，總是寫個一兩篇就無限停更，找不到一個維持住寫作熱情的動力，或許是因為以往開始這件事太輕鬆，找個主流blog網站申請個帳號就可以開始，不花力氣的事就不會太重視，這才是真正的原因，絕對不是我太懶。

以往我用的blog都使用後台管理模式，可以登入帳號密碼到後台，客製化自己blog的版面與新增文章等等基本功能，對於開發者來說用別人的系統多少會有種綁手綁腳的感覺，相較於使用介面，我更喜歡直接面對程式碼，只要我願意花時間去看code，想要什麼功能都能自己想辦法弄出來，這種踏實感在我卡關並懷疑自己是不是時間太多的時候，可以起到很好的激勵效果。

Hugo

Solution

  • Local
    • macOS
    • Typora Editor
    • Git
    • Hugo
  • Site
    • Github
    • Optional
      • domain name registrar
      • ssl

Start

  • Hugo：build from source code
    1. Get source code for github
    2. Add dependence and build
    3. Set path
      1. Determine your default shell
        • echo $SHELL
      2. Edit ~/.zshrc or ~/.bashrc
        • vim ~/.zshrc
        • vim ~/.bashrc
      3. Add hugo path to config file
        • export PATH=$PATH:(your_hugo_path)
      4. Reload
        • sorce ~/.zshrc
        • source ~/.bashrc
    4. Test
      • hugo help
