Service分类

同步/异步 公式 说明
Simple RPC 同步 rpc method_name(message) returns (message) {} 直接调用
Server-side Stream RPC 同步 rpc method_name(message) returns (stream message) {} 服务器通过管道的方式返回
Client-side Stream RPC 异步 rpc method_name(stream message) returns (message) {} 客户端通过管道的方式发送
Bidirectional Stream RPC 异步 rpc method_name(stream message) returns (stream message) {} 客户端和服务器都通过管道的方式发送和返回

调用

根据Service分类的方式不同，服务器端的代码和客户端代码会有所不同，具体如下

Simple RPC

服务器端

    static class SimpleRpcServiceImpl extends SimpleRpcServiceGrpc.SimpleRpcServiceImplBase {
        @Override
        public void service(SimpleRpcIn request,StreamObserver<SimpleRpcOut> responseObserver){
            logger.info("request:" + request.getInmessage());
            SimpleRpcOut simpleRpcOut = SimpleRpcOut.newBuilder().setOutmessage("simple rpc out...").build();
            responseObserver.onNext(simpleRpcOut);
            responseObserver.onCompleted();
        }
    }
  1. 继承自XXXGrpc.XXXImplBase
  2. 实现对应的方法
  3. 调用StreamObserver的onNext方法返回对应对象，对象通过message.newBuilder()….build()获取
  4. 调用StreamObserver的onCompleted方法完成服务器端调用

客户端

        this(ManagedChannelBuilder.forAddress(host,port)
                .usePlaintext(true)
                .build());

...
    
        this.managedChannel = channel;
        blockingStub = SimpleRpcServiceGrpc.newBlockingStub(managedChannel);

...
    
            SimpleRpcIn request = SimpleRpcIn.newBuilder().setInmessage("Simple Rpc In.....").build();
        SimpleRpcOut response;
        try {
            logger.info("start call server....");
            response = blockingStub.service(request);
        } catch (StatusRuntimeException e) {
            logger.log(Level.WARNING, "RPC failed: {0}", e.getStatus());

  1. 通过ManagedChannelBuilder构建managedChannel
  2. 通过ManagedChannel构建BlockingStub（同步调用所以使用BlockingStub）
  3. 构建调用参数，通过BlockingStub调用，并处理返回结果

###Server-side Stream RPC

服务器端

    static class ServerSideStreamServiceImpl extends ServerSideStreamServiceGrpc.ServerSideStreamServiceImplBase {
        @Override
        public void service(ServerSideStreamIn request, StreamObserver<ServerSideStreamOut> responseObserver){
            logger.info("request:" + request.getInmessage());
            List<ServerSideStreamOut> serverSideStreamOutList = new ArrayList<ServerSideStreamOut>();
            for(int i = 0 ; i < 10 ; i++){
                serverSideStreamOutList.add(ServerSideStreamOut.newBuilder().setOutmessage("simple rpc out..." + i).build());
            }

            for(ServerSideStreamOut serverSideStreamOut: serverSideStreamOutList){
                responseObserver.onNext(serverSideStreamOut);
            }
            responseObserver.onCompleted();
        }
    }
  1. 继承自XXXGrpc.XXXImplBase
  2. 实现对应的方法
  3. 调用StreamObserver的onNext方法返回对应对象，因为是管道对象所以可能会调用多次onNext，对象通过message.newBuilder()….build()获取
  4. 调用StreamObserver的onCompleted方法完成服务器端调用

客户端

  • Server-side Stream RPC的客户端调用方式和Simple RPC调用方式相同，也是同步调用

Client-side Stream RPC

服务器端

	    static class ClientSideStreamServiceImpl extends ClientSideStreamServiceGrpc.ClientSideStreamServiceImplBase {
        @Override
        public StreamObserver<ClientSideStreamIn> service(
                final StreamObserver<ClientSideStreamOut> responseObserver) {
            return new StreamObserver<ClientSideStreamIn>() {
                int inCount = 0;
                StringBuilder totalOutSb = new StringBuilder();


                @Override
                public void onNext(ClientSideStreamIn clientSideStreamIn) {
                    inCount++;
                    logger.info("current:" + inCount + ":" + clientSideStreamIn.getInmessage());
                    totalOutSb.append("Client Side Stream Out " + inCount + "\r\n");
                }

                @Override
                public void onError(Throwable throwable) {
                    logger.log(Level.WARNING,"Encountered error in Client Side Stream error:" + throwable.getMessage());
                }

                @Override
                public void onCompleted() {
                    responseObserver.onNext(ClientSideStreamOut.newBuilder().setOutmessage(totalOutSb.toString()).build());
                    responseObserver.onCompleted();
                }
            };
        }
    }
  1. 继承自XXXGrpc.XXXImplBase
  2. 实现对应的方法
  3. 因为客户端是管道的形式，所以需要实现客户端的StreamObserver的onNext, onError, onCompleted方法

客户端

        this(ManagedChannelBuilder.forAddress(host,port)
                .usePlaintext(true)
                .build());

...
    
        this.managedChannel = channel;
        asyncStub = ClientSideStreamServiceGrpc.newStub(managedChannel);

...
        public void serviceClient() throws InterruptedException {
        logger.info("Will try to ClientSideStream  ...");
        final CountDownLatch finishLatch = new CountDownLatch(1);
        StreamObserver<ClientSideStreamOut> responseObserver = new StreamObserver<ClientSideStreamOut>(){

            @Override
            public void onNext(ClientSideStreamOut clientSideStreamOut) {
                logger.info(clientSideStreamOut.getOutmessage());
            }

            @Override
            public void onError(Throwable throwable) {
                Status status = Status.fromThrowable(throwable);
                logger.log(Level.WARNING, "RecordRoute Failed: {0}", status);
                finishLatch.countDown();
            }

            @Override
            public void onCompleted() {
                logger.info("Finished RecordRoute");
                finishLatch.countDown();
            }
        };

        StreamObserver<ClientSideStreamIn> requestObserver = asyncStub.service(responseObserver);

        try{
            Random rand = new Random();
            for(int i = 0 ; i < 10 ; i++){
                ClientSideStreamIn clientSideStreamIn = ClientSideStreamIn.newBuilder().setInmessage("Client Side Stream In:" + i +" ...").build();
                requestObserver.onNext(clientSideStreamIn);
                Thread.sleep(rand.nextInt(1000) + 500);
                if(finishLatch.getCount() == 0){
                    return;
                }
            }

        }catch(RuntimeException e){
            requestObserver.onError(e);
            throw e;
        } catch (InterruptedException e) {
            requestObserver.onError(e);
            throw e;
        }

        requestObserver.onCompleted();

        finishLatch.await(1,TimeUnit.MINUTES);
  1. 通过ManagedChannelBuilder构建managedChannel
  2. 通过ManagedChannel构建Stub（异步调用所以使用Stub）
  3. 客户端调用完成两件事
    • 定义服务器端返回的处理方式，实现StreamObserver的onNext, onError, onCompleted处理
    • 通过Stub调用服务器端的service，取得自己的StreamObserver并将后续消息发送进管道
  4. 调用客户端的onComplete方法
  5. 可以通过CountDownLatch记录当前的状态

Bidirectional Stream RPC

服务器端

    static class BidirectionalServiceImpl extends BidirectionalServiceGrpc.BidirectionalServiceImplBase {
        @Override
        public StreamObserver<BidirectionalIn> service(
                final StreamObserver<BidirectionalOut> responseObserver) {
            return new StreamObserver<BidirectionalIn>() {
                int inCount = 0;

                @Override
                public void onNext(BidirectionalIn bBidirectionalIn) {
                    inCount++;
                    logger.info("current:" + inCount + ":" + bBidirectionalIn.getInmessage());
                    BidirectionalOut bidirectionalOut = BidirectionalOut.newBuilder().setOutmessage("Bidirectional Out message:"+inCount).build();
                    responseObserver.onNext(bidirectionalOut);

                }

                @Override
                public void onError(Throwable throwable) {
                    logger.log(Level.WARNING, "Encountered error in Client Side Stream error:" + throwable.getMessage());
                }

                @Override
                public void onCompleted() {
                    responseObserver.onCompleted();
                }
            };
        }
  1. 继承自XXXGrpc.XXXImplBase
  2. 实现对应的方法
  3. 调用StreamObserver的onNext方法返回对应对象，因为是管道对象所以可能会调用多次onNext，对象通过message.newBuilder()….build()获取
  4. 调用StreamObserver的onCompleted方法完成服务器端调用
